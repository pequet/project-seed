# Seed Project

This repository is a Project-Level Archetype for creating new projects within the TRI Framework, with an emphasis on AI-assisted development. It provides a structured foundation that leverages AI tools and a "memory bank" system to ensure development continuity and efficiency.

### What this is

- **Boilerplate**: A minimal, bare-bones starter kit designed for extensibility.
- **Methodology-first**: Uses frontmatter to encode relationships, not hard-coded links, enabling a flexible knowledge graph.
- **Composable**: Views are computed via DataviewJS and a customJS API (e.g., ConceptManager), allowing for dynamic and maintainable interfaces.

## Getting Started

To start using this Seed Project:

- **Dashboard**: See the [[Dashboard]] for a project overview and entry points.
- **Configuration**: See [[Config.md]] to understand the project's subject, valid filters, and component usage.

### Requirements

This boilerplate is heavily optimized for an AI-assisted development workflow. 

- **AI-Assisted IDE:** An editor like [Cursor](https://cursor.com) or VS Code with GitHub Copilot is recommended.
- **[vibe-tools](https://github.com/eastlondoner/vibe-tools):** Used across rules and scripts and is crucial for many automated tasks.
- **[SpecStory Extension](https://github.com/specstoryai/getspecstory):** Helps with long-term memory by saving AI interactions.
- **Memory Bank:** The project's core long-term memory system resides in the `memory-bank/` directory.

## Core ideas

- **Single source of truth**: Each concept has one file and one structural location.
- **Functional roles via metadata**: Use `domain-category` to indicate roles (e.g., `person`, `topic`, `event`).
- **Group relationships**: Express connections with `group-[type]` fields (e.g., `group-department`, `group-status`).
- **Dynamic views**: All views are generated by reusable components that read from the metadata, ensuring that views update automatically as the data changes.

## Master Rules & IDE Synchronization

This boilerplate includes a `master-rules/` directory containing the archetypal, source-of-truth AI rules/instructions. Use your [IDE Rules Synchronizer](https://github.com/pequet/ide-rules-synchronizer) script to generate assistant-specific outputs on demand (e.g., `.cursor/rules/`, `.github/instructions/`, `CLAUDE.md`). These generated files are not included by default.

Example usage:
```bash
./sync-ide-rules.sh /path/to/your/project
```

## File layout (key paths)

- `Dashboard.md`: Project overview and entry point.
- `Config.md`: Project subject, valid filters, and component references.
- `Models/`: Contains all project-specific data and knowledge.
- `Views/`: Contains all hubs, dashboards, and visual representations of the knowledge.
- `Controllers/`: Contains all methods and processes.
- `memory-bank/`: Contains the project's long-term memory and context for AI.
- `master-rules/`: Contains the source-of-truth rules for AI behavior.

## Recommended Patterns

While this seed is open-ended, it is designed to easily evolve towards the patterns used in the `project-sample`. Consider the following recommendations:

- **Authoring:** Use `domain: concepts` for primary knowledge items and `domain-category` in hub/group files to define their role. Add group affiliations with `group-[type]: value`.
- **Rendering:** For dynamic views, the framework assumes an environment like Obsidian with the Dataview plugin and a customJS environment providing a `ConceptManager` API or similar. See `Config.md` for examples.
